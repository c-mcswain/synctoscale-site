Quickstart

Get up and running with SyncToScale in minutes. This guide walks through creating a connection, triggering a workflow, and verifying execution.

HTTP Methods at a Glance

POST Sends data to create, run, or process something
GET Retrieves information without changing anything

Most examples below show how SyncToScale receives data or triggers workflows programmatically.

Step 1: Create a Connection

Start by authenticating an external system (such as an HRIS, CRM, or messaging platform). Connections are securely stored and reused across workflows. 

POST /connections
{
  "provider": "slack",
  "auth_type": "oauth",
  "scopes": ["chat:write", "users:read"]
}

Step 2: Configure a Trigger

Triggers determine when a workflow starts. Event-based triggers activate immediately when new data enters the system. 

POST /triggers/event
{
  "provider": "bamboohr",
  "event_type": "employee.created"
}

Step 3: Run the Workflow

Once configured, workflows can be triggered automatically or executed manually for testing. 

POST /workflows/{id}/run
{
  "payload": {
    "employee_id": "12345",
    "email": "newhire@example.com"
  }
}

Step 4: Verify Execution

Use logs to confirm successful execution and review any warnings or errors. 

GET /runs/{run_id}
{
  "status": "success",
  "completed_at": "2025-01-20T14:42:00Z"
}

From here, you can extend workflows with actions, mappings, and Nova-powered intelligence.
