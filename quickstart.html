Quickstart

Get your first SyncToScale workflow running in minutes. This guide walks through auth setup, triggering a run, and checking results with simple API examples you can copy and paste.

HTTP Methods at a Glance

POST Sends data to create, run, or process something
GET Retrieves information without changing anything

Most examples below show how SyncToScale receives data or triggers workflows programmatically.

1) What you’ll need

  • Your SyncToScale workspace (demo environment is fine)
  • An API key
  • A workflow ID (or a starter workflow you can trigger)
Tip: If you’re just validating the experience for a recruiter-ready walkthrough, a single “Hello World” workflow that logs payload fields is enough to demonstrate end-to-end behavior.

2) Set up authentication

SyncToScale uses a simple API key sent as a bearer token. Keep keys server-side and rotate regularly. 

Authorization: Bearer YOUR_API_KEY

3) Trigger your first workflow run

Use the run endpoint to send a payload that SyncToScale can process. You’ll receive a run_id you can use to check status.

POST Run Workflow

curl -X POST "https://api.synctoscale.me/v1/workflows/run" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "workflow_id": "wf_12345",
    "source": "quickstart",
    "payload": {
      "employee_id": "E-1042",
      "event": "new_hire_created",
      "timestamp": "2025-12-13T12:00:00Z"
    }
  }'
Replace wf_12345 with your workflow ID. Your payload can include any fields your workflow expects.

4) Check run status

Once you have a run_id, you can fetch status and results without triggering anything again.

GET Get Run Status

curl -X GET "https://api.synctoscale.me/v1/runs/run_abc123" \
  -H "Authorization: Bearer YOUR_API_KEY"

Example response: 

{
  "run_id": "run_abc123",
  "status": "completed",
  "started_at": "2025-12-13T12:00:03Z",
  "completed_at": "2025-12-13T12:00:06Z",
  "workflow_id": "wf_12345",
  "summary": {
    "steps_total": 6,
    "steps_succeeded": 6,
    "steps_failed": 0
  }
}
