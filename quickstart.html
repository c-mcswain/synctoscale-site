Quickstart

Get up and running with SyncToScale in minutes. This guide walks through authenticating, creating a connection, and running your first workflow end-to-end.

Step 1 ‚Äî Authenticate

All SyncToScale API requests require a project token. Include it in the Authorization header. 

Authorization: Bearer <project_token>
Content-Type: application/json

POST Create a Connection

Connections link SyncToScale to external systems like HRIS, CRMs, and messaging tools. 

POST /connections
{
  "provider": "slack",
  "auth_type": "oauth",
  "scopes": ["chat:write", "users:read"]
}

POST Run a Workflow

Trigger a workflow manually by sending a payload to the workflow run endpoint. 

POST /workflows/{id}/run
{
  "payload": {
    "employee_id": "12345",
    "email": "test@example.com"
  }
}

GET Check Run Status

Retrieve execution details and confirm whether the workflow completed successfully. 

GET /runs/{run_id}
{
  "run_id": "run_789",
  "status": "completed",
  "completed_at": "2025-01-20T14:05:12Z"
}
