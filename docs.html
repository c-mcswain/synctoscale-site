Developer Documentation

Everything you need to integrate SyncToScale into your stack.

Platform Overview

SyncToScale is an automation orchestration layer that connects SaaS tools like Salesforce, HubSpot, Slack, Jira, and Google Workspace. Developers can trigger workflows, listen to webhook events, and sync objects across platforms in real time.

Getting Started

  1. Create a SyncToScale developer account.
  2. Generate your API key under Developer → Tokens.
  3. Install the SyncToScale client in your environment.
  4. Authenticate your connected apps (Salesforce, Slack, etc.).
  5. Start triggering automation workflows.

// Example: Node.js client initialization
import SyncClient from "@synctoscale/client";

const client = new SyncClient({
  apiKey: process.env.SYNC_API_KEY
});

client.trigger("lead.created", {
  source: "salesforce",
  payload: {...}
});

Webhook Events

SyncToScale emits webhook events when connected platforms update their data.

  • lead.created
  • contact.updated
  • deal.stage_changed
  • user.permission_updated

Each webhook contains:

  • event – event name
  • timestamp – ISO-8601 timestamp
  • source – platform (Salesforce, Slack, Google Workspace)
  • payload – record data

Authorization

All API requests must include your bearer token:


curl -X GET https://api.synctoscale.me/v1/users \
  -H "Authorization: Bearer <API_KEY>"

API keys can be rotated at any time without impacting existing webhooks.
