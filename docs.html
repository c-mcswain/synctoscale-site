Developer Documentation

A lightweight API designed for workflow automation, event handling, and two-way data syncing. This reference describes endpoints, objects, and concepts used throughout SyncToScale.

Authentication

SyncToScale uses static API keys to authenticate requests. Keys can be generated under: Settings → Developer. 


Authorization: Bearer YOUR_API_KEY

Example:


curl -X GET "https://api.synctoscale.me/v1/workflows" \
  -H "Authorization: Bearer sk_test_12345"

API Endpoints


GET /v1/workflows

Response:

{
  "data": [
    { "id": "wf_1", "name": "CRM Sync", "status": "active" },
    { "id": "wf_2", "name": "New Hire Flow", "status": "inactive" }
  ]
}


GET /v1/workflows/{workflow_id}


POST /v1/workflows/{workflow_id}/run


{
  "status": "queued",
  "run_id": "run_44c98a"
}


GET /v1/workflow-runs?workflow_id=wf_1

Webhooks

SyncToScale can send events to your webhook URL for real-time notifications.


POST https://yourapp.com/webhooks/synctoscale

Example Payload:

{
  "event": "workflow.completed",
  "run_id": "run_44c98a",
  "workflow_id": "wf_7d92ab",
  "success": true,
  "duration_ms": 8134
}

Errors


400  Bad Request — Invalid payload
401  Unauthorized — Invalid or missing API key
403  Forbidden — Your key has insufficient scopes
404  Not Found — Workflow or object does not exist
429  Rate Limit Exceeded
500  Server Error — Retry with backoff

Rate Limits


1000 requests / day (Free Tier)
10 requests / second (All Tiers)
Burst capacity: 30 requests
Retry-After header included when limiting
