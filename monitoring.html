Logs & Monitoring

SyncToScale provides real-time visibility into workflow execution, connection health, payload transformations, and Nova-assisted anomaly detection—helping you diagnose issues quickly.

Overview

Monitoring is essential for troubleshooting automation issues. SyncToScale captures every workflow run, each step execution, payload inputs/outputs, connection behavior, and warning or error states.

Workflow Run History

Every workflow execution is logged with timestamps, trigger details, and run-level metadata. 

GET /workflows/{id}/runs
[
  {
    "run_id": "run_123",
    "status": "success",
    "trigger": "employee.created",
    "started_at": "2025-01-20T14:00:01Z",
    "duration_ms": 420
  },
  {
    "run_id": "run_122",
    "status": "failed",
    "trigger": "employee.created",
    "started_at": "2025-01-20T13:58:10Z",
    "duration_ms": 812
  }
]

Failed runs automatically surface their root cause and failing step.

Step-Level Logs

Each action inside the workflow produces its own log entry with full detail. 

{
  "step": "search_user",
  "provider": "google_workspace",
  "status": "success",
  "attempts": 1,
  "input": { "email": "jordan@example.com" },
  "output": { "user_id": "gw_8871" },
  "timestamp": "2025-01-20T14:12:04Z"
}

Logs include the payload before and after transformations for transparent debugging.

Connection Health Monitoring

Every workflow run includes a health snapshot of the connection used: 

{
  "connection_id": "conn_abc123",
  "status": "healthy",
  "rate_limit_remaining": 47,
  "token_expires_in": 3600,
  "last_validated": "2025-01-20T14:00:00Z"
}
  • Low rate limits trigger warnings
  • Expired tokens place the connection into “Needs Attention”
  • Provider issues are surfaced before workflow failure

Payload Snapshots

SyncToScale captures snapshots of the payload at each major stage:

  • Incoming trigger payload
  • Normalized object
  • Mapped payload
  • Provider response
{
  "stage": "mapped_payload",
  "data": {
    "full_name": "Jordan Lee",
    "email": "jordan@example.com",
    "department": "Engineering"
  }
}

Error Grouping

When multiple failures share a root cause—permissions, invalid mappings, outages— SyncToScale groups them automatically for easier diagnosis. 

{
  "group": "insufficient_permissions",
  "occurrences": 14,
  "affected_workflows": ["new-hire-sync", "it-provisioning"],
  "recommended_fix": "Re-authenticate Slack with chat:write scope"
}

Grouped insights help identify systemic issues quickly.

Nova Anomaly Detection

Nova continuously analyzes workflow runs and connection behavior to detect anomalies such as:

  • Spike in failures for a specific action
  • Drift in payload structure
  • Rate limit saturation
  • Unexpected increases in duration
  • Provider instability
Nova Insight:
{
  "anomaly": "Error rate increased 4x in past 30 minutes",
  "likely_cause": "HubSpot API latency",
  "confidence": 0.92
}
