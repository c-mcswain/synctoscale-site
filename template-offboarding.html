Employee Offboarding

Template

Standardize employee offboarding across HR, IT, and security. Automatically disable access, collect assets, and log an auditable trail whenever someone leaves your company.

Category: HR & IT Security Difficulty: Intermediate Trigger: Termination event Systems: HRIS ¬∑ Identity ¬∑ Ticketing ¬∑ Slack

Overview

The Employee Offboarding template coordinates secure offboarding whenever an employee is terminated or leaves the organization. Once HR triggers a termination in your HRIS, SyncToScale disables accounts, revokes group access, opens IT and security tickets, and posts a clear summary to the right internal channels ‚Äî while Nova checks for lingering access or high-risk assets that still need attention.

When to use this template

  • You want a consistent, auditable offboarding checklist for every departing employee.
  • IT is manually deprovisioning dozens of apps when someone leaves.
  • Security wants proof that access was removed promptly and fully.
  • You need offboarding to work for both voluntary and involuntary terms.

What this template automates

  • Detects termination events in the HRIS.
  • Disables core identity (Google Workspace / M365) and revokes SSO groups.
  • Creates IT and Security tickets with a checklist of follow-up actions.
  • Notifies managers and People Ops with a summary of what was deprovisioned.
  • Logs all actions for audit and access review reports.

How it works

  1. HR updates the employee‚Äôs status in the HRIS to a termination status.
  2. SyncToScale receives the termination event or picks it up on a short interval sync.
  3. Nova validates key fields (last day, termination type, manager, location).
  4. Core identity is disabled and high-risk apps are deprovisioned first.
  5. IT and Security tickets are created to collect equipment and close any remaining access.
  6. Slack notifications summarize offboarding, highlighting any manual follow-ups.

Workflow steps

  1. Step 1 ‚Äî Trigger: Termination in HRIS

    The workflow fires when an employee record is updated with employmentStatus = "Terminated" or a similar configured status (e.g., "Voluntary", "Involuntary", "End of contract").

  2. Step 2 ‚Äî Validate offboarding data

    SyncToScale pulls termination details and uses Nova to confirm that required fields are present: last working day, manager, location, and termination reason.

    Behind the scenes ‚Äî Nova pre-checks

    • Flags missing last day or conflicting end dates across systems.
    • Detects if the user is already inactive in identity but active in HRIS, or vice versa.
    • Highlights contractors or temporary workers who may be managed in different tools.

  3. Step 3 ‚Äî Disable core identity & SSO

    SyncToScale disables the user in Google Workspace or Microsoft 365 and revokes Okta / SSO groups. High-risk apps (admin tools, finance, production access) are prioritized first.

  4. Step 4 ‚Äî Revoke app access

    Based on the user‚Äôs groups and application assignments, the template deprovisions collaboration, CRM, engineering, and other SaaS tools using your identity or direct connectors.

  5. Step 5 ‚Äî Create IT & Security tickets

    A combined IT/Security offboarding ticket is created (or two separate tickets, depending on configuration) with a checklist for hardware return, forwarding rules, and any special data handling requirements.

  6. Step 6 ‚Äî Notify stakeholders

    SyncToScale posts a summary to Slack (e.g., #offboarding) and optionally DMs the manager and People Ops with the status of access removal and open tasks.

Error handling

Offboarding should never silently fail. This template includes explicit guardrails for partial failures and missing data.

  • Identity disable failures immediately create a high-priority Security ticket.
  • SSO deactivation issues are retried and logged as blocking until resolved.
  • Missing termination date or status create a ‚ÄúNeeds HR review‚Äù task instead of running offboarding.
  • All errors are visible in Nova‚Äôs run history with next-best actions.

Set up this template in SyncToScale

  1. Open Templates > Library and select Employee Offboarding.
  2. Connect your HRIS, identity provider, SSO, ticketing, and Slack tools.
  3. Configure which termination statuses should trigger automatic offboarding.
  4. Define your list of high-risk apps and groups for prioritized removal.
  5. Map HRIS fields to the template‚Äôs required termination fields.
  6. Run in Dry Run for a few terminations, then switch to Active.
