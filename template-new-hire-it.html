New Hire IT Onboarding

Template

Automatically provision accounts, groups, and tickets when a new employee is created in your HRIS. Keep IT, HR, and managers in lockstep from offer accept to day one.

Category: HR & IT Ops Difficulty: Intermediate Trigger: New hire created Systems: HRIS · Google Workspace · Okta · Jira · Slack

Overview

The New Hire IT Onboarding template listens for a new employee record in your HRIS (e.g., BambooHR, Rippling, Gusto, HiBob) and orchestrates a standardized IT onboarding flow. SyncToScale generates usernames, requests hardware, creates accounts, and notifies the right people — while Nova watches for incomplete data and potential conflicts.

When to use this template

  • You want a consistent, auditable onboarding checklist for every new hire.
  • IT is manually creating accounts in multiple tools today.
  • Start dates are slipping because access isn’t ready on day one.
  • You want Nova to flag missing manager, location, or role data before provisioning.

What this template automates

  • Create core accounts (Google Workspace / Microsoft 365).
  • Assign SSO / Okta groups based on department and role.
  • Create an IT onboarding ticket in Jira or ServiceNow.
  • Post a summary to an #it-new-hires Slack channel.
  • Optionally request laptop and hardware.

How it works

  1. HR creates a new employee in the HRIS with status Pending or Hired.
  2. HRIS sends a webhook to SyncToScale (or is polled on a schedule).
  3. SyncToScale normalizes the payload, generates a username, and enriches department + manager data.
  4. Accounts and groups are created across identity, collaboration, and ticketing tools.
  5. Nova validates required fields and surfaces any issues as guided fixes.
  6. A single Slack summary message confirms successful provisioning with deep links to each system.

Workflow steps

  1. Step 1 — Trigger: New hire created in HRIS

    The workflow is triggered when a new employee is created with a hireDate in the future and employmentStatus = "Hired".

  2. Step 2 — Normalize and validate profile

    SyncToScale normalizes names, generates a proposed email username, and validates that managerEmail, location, and jobTitle are present.

    Behind the scenes — Nova validation

    • Suggests alternate usernames if email is already taken.
    • Flags start dates in the past or more than 90 days in the future.
    • Highlights missing manager or department for IT follow-up.

  3. Step 3 — Create core identity account

    Create a Google Workspace (or Microsoft 365) account with the normalized profile and assign the appropriate license SKU.

  4. Step 4 — Assign Okta / SSO groups

    Based on department and location, SyncToScale assigns SSO groups that map to standard app bundles (e.g., “Sales – US”, “Engineering – EU”).

  5. Step 5 — Create IT onboarding ticket

    A Jira / ServiceNow ticket is created with a checklist of tasks (hardware, access, security training) and is automatically assigned to the IT onboarding queue.

  6. Step 6 — Notify stakeholders

    A summary is posted to Slack (e.g., #it-new-hires) with links to the HRIS profile, identity admin panel, and Jira ticket.

Field mappings

HRIS → Identity (Google Workspace)

Source (HRIS) Transform Target (Identity)
firstName Capitalize name.givenName
lastName Capitalize name.familyName
preferredName (fallback: firstName) Use if present name.fullName
department Normalize to canonical list organizations[0].department
jobTitle Trim and title-case organizations[0].title
location Map to office / region code addresses[0].country, addresses[0].locality
managerEmail Lookup user ID relations[0].value (manager)
workEmail (generated if missing) {{firstName[0]}}{{lastName}}@company.com primaryEmail

HRIS → Okta group assignment

Rule Condition Assigned group(s)
Sales – US department = "Sales" and location.country = "US" grp_sales_us_core, grp_crm_users
Engineering – EU department = "Engineering" and location.region = "EU" grp_eng_eu_core, grp_git_repo_access
HR – Global department = "People" or "HR" grp_hr_global, grp_hris_admin

Error handling

This template includes opinionated error handling that helps IT fix issues quickly without replaying the entire workflow manually.

  • Identity creation errors are retried up to 3 times with exponential backoff.
  • Conflicting email addresses raise a username_conflict error with Nova suggestions.
  • Missing required HR fields create a “Blocked” Jira issue assigned to HR or People Ops.
  • All errors are summarized in Nova’s run history and can be exported as CSV.

Nova insights for this template

  • Detects conflicting email addresses and suggests safe alternatives.
  • Flags missing manager, department, or location fields before provisioning.
  • Highlights hires starting in < 48 hours with incomplete access setup.
  • Shows a grounded timeline of which systems were provisioned in which order.
  • Surfaces long-running steps (e.g., hardware shipping) in a single view.

Set up this template in SyncToScale

  1. Go to Templates > Library and select New Hire IT Onboarding.
  2. Connect your HRIS, identity provider (Google / M365), Okta, Jira, and Slack.
  3. Review and adjust the username generation rules and Okta group mapping table.
  4. Map HRIS fields to the template’s required fields using the visual mapper.
  5. Turn on Nova validation for “Required onboarding data”.
  6. Run in Dry Run mode for 1–2 hires, then switch the toggle to Active.
