Nova-Assisted Cleanup

Template

Use Nova to scan your data for duplicates, inconsistent values, and obvious errors across HRIS, CRM, and support tools — then generate safe, reviewable cleanup proposals you can apply with one click.

Category: Developer & Custom Difficulty: Intermediate Trigger: Manual or scheduled health check Systems: HRIS · CRM · Support · Identity · Warehouse

Overview

The Nova-Assisted Cleanup template is your “data hygiene co-pilot.” It periodically reviews records across systems for common issues — like duplicate contacts, conflicting job titles, invalid emails, or missing IDs — and then builds a cleanup plan. Nova groups related records, suggests the correct source of truth, and generates concrete “fix” operations that can be reviewed and applied safely.

When to use this template

  • Your CRM has many duplicates or half-complete records.
  • HRIS and identity platforms disagree on job titles or departments.
  • Support and CS tools use slightly different account naming schemes.
  • You want an ongoing, low-friction way to keep data clean.

What this template automates

  • Sampling and scanning records for hygiene issues.
  • Grouping potential duplicates or conflicting sets.
  • Proposing specific field updates, merges, or normalizations.
  • Creating review tasks or “cleanup boards” for admins.
  • Applying approved cleanup operations via API in bulk.

How it works

  1. SyncToScale pulls a sample or full set of records from one or more systems.
  2. Nova evaluates patterns and identifies probable issues.
  3. Records are grouped into “cleanup candidates” (e.g., duplicates, mismatches, gaps).
  4. Nova produces proposed updates, merges, or standardizations.
  5. Admins review proposals in your chosen system or cleanup board.
  6. Approved proposals are executed via bulk update API calls.

Workflow steps

  1. Step 1 — Trigger: Manual run or scheduled

    You can run Nova-Assisted Cleanup:

    • On-demand (button click in SyncToScale), or
    • On a recurring schedule (e.g., weekly data hygiene job).

  2. Step 2 — Fetch candidate records

    SyncToScale queries:

    • HRIS employees
    • CRM leads/contacts/accounts
    • Support users or organizations
    • Any other connected dataset you include

    Filters can be used to limit scope (e.g., updated in last 90 days, or only specific segments).

  3. Step 3 — Nova analysis

    Nova runs multiple passes to identify:

    • Likely duplicate records (similar names, emails, domains).
    • Conflicting values for the same entity (e.g., two titles for one person).
    • Missing or invalid critical fields (IDs, emails, dates).
    • Non-standard variants of fields (e.g., “People Ops” vs “People Operations”).

  4. Step 4 — Generate cleanup proposals

    For each candidate, Nova suggests operations such as:

    • Merge record B into A, keep specific fields from each.
    • Normalize a field to a canonical value (department names, countries, statuses).
    • Fill missing values using other systems as reference (e.g., HRIS → CRM).
    • Flag suspicious values for manual review only.

  5. Step 5 — Create review tasks

    Cleanup proposals are written to:

    • A “Data Cleanup” project/board in your task tool, or
    • A special review dashboard inside your CS/RevOps platform.

  6. Step 6 — Apply approved fixes

    Once approved by admins, SyncToScale executes the cleanup operations via API calls in safe batches.

Cleanup proposals format

Proposal Field Description Example
proposalId Unique ID per cleanup suggestion "fix_01J2NOVA001"
type Operation type (MERGE, UPDATE, NORMALIZE, FLAG_ONLY) "MERGE"
targets Records that would be affected ["0035g00000CN456", "0035g00000CN457"]
changes Field-level operations being proposed {"Title": "Director of People Operations"}
novaRationale Why Nova suggests this change "Title normalized based on majority + HRIS source of truth."

Error handling

  • Proposals are never applied automatically in “Preview” mode.
  • All applied changes are wrapped in transactions where supported.
  • Failed operations are rolled back or re-queued when possible.
  • Nova logs confidence levels; low-confidence proposals can require manual approval.

Example safety guard

{
  "runId": "run_01J2NOVACLEAN",
  "status": "halted",
  "reason": "Low-confidence proposals exceed threshold.",
  "novaMessage": "Too many uncertain merges detected. Recommend manual review before proceeding."
}

Nova insights for this template

  • Shows which systems generate the most dirty data.
  • Highlights common sources of duplicates and mismatches.
  • Suggests standardized lists for departments, titles, and statuses.
  • Tracks cleanup impact over time (e.g., duplicate rate down 30%).

Set up this template

  1. Select Nova-Assisted Cleanup from the Templates Library.
  2. Connect the systems you want Nova to analyze (HRIS, CRM, Support, Identity, etc.).
  3. Choose whether to run in Preview-only or Preview + Apply mode.
  4. Configure which entity types to scan (people, accounts, departments, etc.).
  5. Set approval rules (who can approve which changes).
  6. Run your first cleanup and review Nova’s proposals.

Template summary

  • Trigger: Manual or scheduled health check

Best practice defaults

  • Start in Preview-only mode for 1–2 cycles.
  • Require human approval for MERGE operations.
  • Limit scope initially to a single data domain (e.g., CRM contacts).
