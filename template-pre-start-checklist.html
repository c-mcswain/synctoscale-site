Pre-Start Checklist

Template

Automatically spin up a pre-start checklist for every new hire. Make sure hardware, access, paperwork, and welcome moments are ready before day one — with Nova highlighting gaps in real time.

Category: Employee Onboarding Difficulty: Beginner Trigger: New hire created / X days before start Systems: HRIS · Task/Project Tool · Slack · Email

Overview

The Pre-Start Checklist template ensures critical onboarding steps are completed before a new hire’s first day. When a new employee is created in your HRIS (or a certain number of days before their start date), SyncToScale automatically creates a checklist of tasks for People Ops, IT, and the hiring manager — with Nova flagging missing details that could delay onboarding.

When to use this template

  • New hires arrive without laptops, access, or schedules ready.
  • Onboarding tasks live in scattered spreadsheets or docs.
  • You want consistent pre-start steps across departments.
  • You want Nova to warn you before a first-day surprise.

What this template automates

  • Creates a pre-start checklist per new hire in your task/project tool.
  • Assigns tasks to IT, People Ops, and the hiring manager.
  • Sets due dates relative to the hire’s start date.
  • Posts a summary to Slack or email with the checklist link.
  • Uses Nova to highlight gaps (no laptop, missing address, etc.).

How it works

  1. A new hire is created in the HRIS with a start date.
  2. SyncToScale builds a structured onboarding profile.
  3. Nova checks for missing key fields and potential blockers.
  4. A pre-start checklist is created using your chosen template in Asana, ClickUp, Jira, etc.
  5. Tasks are assigned with due dates based on the start date (e.g., T-10 days, T-5 days).
  6. A Slack or email notification shares the checklist with stakeholders.

Workflow steps

  1. Step 1 — Trigger: New hire created / pre-start window

    The workflow can run in two ways:

    • Immediately when a new hire is created (employee.created event), or
    • On a daily schedule that looks for hires whose start date is within a pre-start window (e.g., 7 or 10 days away).

  2. Step 2 — Build onboarding profile

    SyncToScale maps HRIS data into a normalized profile used across your onboarding templates (name, role, location, manager, start date, employment type, etc.).

  3. Step 3 — Nova pre-start validation

    Nova evaluates whether you have enough information to start pre-day-one work:

    • Missing or inconsistent start date.
    • No manager or manager email.
    • Missing location (for equipment shipping or in-office prep).
    • No hardware type selected (laptop, peripherals).
    • Short pre-start window (e.g., start date is < 3 days away).

  4. Step 4 — Create checklist in task/project tool

    SyncToScale creates a project/board or checklist for the new hire using your base template, then fills in details like name, start date, and role-specific tasks.

  5. Step 5 — Assign tasks & due dates

    Tasks are assigned to People Ops, IT, and the manager with due dates relative to the start date. For example:

    • T-10 days: Submit hardware order.
    • T-7 days: Confirm access & systems.
    • T-3 days: Send welcome email + schedule intro meetings.

  6. Step 6 — Notify stakeholders

    A Slack message or email is sent to the manager and People Ops with:

    • Checklist link
    • Summary of key tasks & due dates
    • Nova insights about possible pre-start issues

Field mappings

HRIS → Pre-Start Checklist

HRIS Field Transform Checklist Field
firstName, lastName Combine, title case Checklist title: Pre-Start – {{FullName}}
hireDate Normalize to ISO + timezone Base date for all due dates
department Normalize to static list Used to pick appropriate checklist variant
jobTitle Title case Shown in project custom field
managerEmail Directory lookup Assigned owner / watcher
employmentType (e.g., FTE, Contractor) Map to category Adjusts tasks (e.g., equipment vs. BYOD)

Error handling

Pre-start work is time-sensitive. This template surfaces any blockers early so they can be fixed manually if needed.

  • Missing start date: Nova blocks checklist creation and alerts People Ops.
  • Unknown manager: checklist is still created, but manager-owned tasks are assigned to People Ops until resolved.
  • Project/tool API errors: retries with backoff, then logs a warning and sends a fallback email summary.

Sample diagnostic for missing start date

{
  "runId": "run_01J2PRESTART",
  "status": "pending_information",
  "missingFields": ["hireDate"],
  "novaMessage": "Cannot generate pre-start checklist without a start date. Please update the hireDate in the HRIS."
}

Nova insights for this template

  • Shows which departments consistently cut pre-start timelines too close.
  • Highlights tasks that frequently miss their due dates.
  • Identifies patterns like “remote hires without hardware requests”.
  • Helps People Ops refine pre-start SLAs and expectations.

Set up this template in SyncToScale

  1. Select Pre-Start Checklist from the Templates library.
  2. Connect your HRIS and preferred task/project tool (Asana, ClickUp, Jira, etc.).
  3. Define your pre-start window (e.g., “7 days before start date”).
  4. Customize the task list for your organization and departments.
  5. Turn on Nova’s pre-start validation checks.
  6. Run a test with a mock new hire, then enable in Active mode.

Best practice defaults

  • Use at least a 7–10 day pre-start window.
  • Keep 8–12 core tasks that apply to every hire.
  • Add department-specific tasks via rules, not manual edits.

Nova will help you…

  • Spot hires at risk of a rocky first day.
  • See which teams are consistently late on pre-start tasks.
  • Refine your checklist to prevent recurring issues.
