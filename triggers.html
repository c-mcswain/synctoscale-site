Triggers

Triggers determine when a workflow starts. SyncToScale supports event-based, polling, scheduled, and webhook triggers to activate workflows exactly when new data enters your system.

Overview

A trigger is the entry point of a workflow. When a trigger fires, SyncToScale collects the incoming payload, validates the structure, and passes it into the first mapping or action step.

POST Event-Based Triggers (Recommended)

Event-based triggers activate immediately when the connected system generates an event such as a new employee record or updated deal. 

POST /triggers/event
{
  "provider": "bamboohr",
  "event_type": "employee.created",
  "payload": {
    "id": "98765",
    "first_name": "Jordan",
    "email": "jordan@example.com"
  }
}

These triggers are the most reliable and are recommended for real-time automations.

GET Polling Triggers

For systems without native events, SyncToScale can poll an endpoint at a configurable interval to detect new or changed records. 

GET /polling
{
  "provider": "legacy_hris",
  "last_checked": "2025-01-20T12:00:00Z"
}
  • Delta detection (created/updated since last run)
  • Deduplication safeguards
  • Backoff handling when provider is throttled
  • Recovery from missed runs due to outages

POST Scheduled Triggers

Scheduled triggers run workflows on a fixed cadence (hourly, daily, weekly) using validated cron expressions. 

POST /triggers/schedule
{
  "cron": "0 * * * *",
  "timezone": "UTC"
}

SyncToScale prevents overlapping runs and enforces safe execution windows.

POST Webhook Triggers

Custom applications can send data directly into a workflow using secure, signed inbound webhooks. 

POST /webhooks/inbound
Headers:
  X-Signature: sha256=fc1db1...

Body:
{
  "type": "employee.updated",
  "data": {
    "id": "12345",
    "email": "new@example.com"
  }
}

SyncToScale verifies signatures, timestamps, and replay protection before accepting a webhook.
