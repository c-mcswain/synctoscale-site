Triggers

Triggers determine when a workflow starts. SyncToScale supports event-based, polling, and scheduled triggers, ensuring workflows activate exactly when new data enters your system.

Overview

A trigger is the entry point of a workflow. When the trigger fires, SyncToScale collects the incoming payload, validates the structure, and passes it into the first mapping/action step.

Event-Based Triggers (Recommended)

Event-based triggers activate immediately when the connected system generates an event — such as a new employee record, updated deal, or form submission. 

POST /triggers/event
{
  "provider": "bamboohr",
  "event_type": "employee.created",
  "payload": {
    "id": "98765",
    "first_name": "Jordan",
    "email": "jordan@example.com"
  }
}

These triggers are the most reliable and are recommended for real-time automations.

Polling Triggers

Some systems do not support outgoing event notifications. SyncToScale can poll an endpoint at a configurable interval (default: every 5 minutes), detecting new or changed records. 

GET /polling
{
  "provider": "legacy_hris",
  "last_checked": "2025-01-20T12:00:00Z"
}

Polling logic includes:

  • Delta detection (created/updated since last run)
  • Deduplication safeguards
  • Backoff handling when provider is throttled
  • Recovery from missed runs due to outages

Scheduled Triggers

These allow workflows to run on a fixed cadence (hourly, daily, weekly). Useful for reconciliations, audits, batch exports, usage reports, and overnight syncs. 

POST /triggers/schedule
{
  "cron": "0 * * * *",
  "timezone": "UTC"
}

SyncToScale uses safe cron validation and prevents overlapping runs.

Webhook Triggers

Custom applications can send data directly into a workflow using secure, signed webhooks. 

POST /webhooks/inbound
Headers:
  X-Signature: sha256=fc1db1...

Body:
{
  "type": "employee.updated",
  "data": {
    "id": "12345",
    "email": "new@example.com"
  }
}

SyncToScale verifies signatures, timestamps, and replay protection before accepting a webhook.

Troubleshooting Triggers

  • Duplicate Events — Occurs when upstream systems retry event delivery.
  • Missing Events — Provider outage or webhook misconfiguration.
  • Invalid Payloads — Schema mismatch; transformation required.
  • Timezone Drift — Scheduled triggers running at incorrect hours.
  • Polling Overload — Provider rate limits exceeded; backoff enabled.

Nova can analyze these patterns to provide automatic recommendations.
