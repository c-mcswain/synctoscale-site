Nova AI API

Nova is SyncToScale’s AI engine for field mapping, anomaly detection, error classification, and workflow recommendations. Use the Nova API to make your automations smarter and more resilient.

Overview

Nova analyzes payloads, schemas, connection states, and historical workflow runs to:

  • Suggest field mappings between systems
  • Detect anomalies and spikes in failure rates
  • Classify errors into actionable categories
  • Recommend transformations and default values
  • Surface likely root cause for sync issues

Authentication

The Nova API uses the same project-level authentication as the core SyncToScale API. You must include a valid API key or OAuth token in each request. 

GET /nova/v1/health
Authorization: Bearer <project_token>

{
  "status": "ok",
  "model": "nova-sync-1.0",
  "uptime_seconds": 87423
}

Field Mapping Suggestions

Nova can propose mappings between a source payload and a target schema. This is especially helpful during implementation or when working with unfamiliar systems. 

POST /nova/v1/mapping/suggest
{
  "source_payload": {
    "fname": "Jordan",
    "lname": "Lee",
    "mail": "jordan@example.com"
  },
  "target_schema": {
    "first_name": "string",
    "last_name": "string",
    "email": "string"
  }
}

Response:
{
  "suggested_mapping": {
    "first_name": "fname",
    "last_name": "lname",
    "email": "mail"
  },
  "confidence": {
    "first_name": 0.97,
    "last_name": 0.96,
    "email": 0.99
  }
}

Implementers can accept, edit, or override these mappings.

Anomaly Detection

Nova monitors workflow runs and connection behavior to detect outliers and patterns. 

POST /nova/v1/anomalies/detect
{
  "window_minutes": 30,
  "workflow_id": "new-hire-sync"
}

Response:
{
  "anomalies": [
    {
      "type": "error_rate_spike",
      "severity": "high",
      "metric": "failure_rate",
      "previous_baseline": 0.02,
      "current_value": 0.18
    }
  ],
  "suggested_actions": [
    "Check recent changes to mappings",
    "Review HubSpot connection health",
    "Inspect provider status page"
  ]
}

Error Classification

Rather than leaving teams to interpret raw error messages, Nova classifies errors into consistent categories with recommended fixes. 

POST /nova/v1/errors/classify
{
  "provider": "slack",
  "http_status": 403,
  "raw_error": "missing_scope: chat:write"
}

Response:
{
  "category": "insufficient_permissions",
  "likely_cause": "Missing chat:write scope",
  "recommended_fix": "Re-authenticate Slack with the required scope",
  "confidence": 0.94
}

Transformation Recommendations

Nova can propose transformations when data types or formats don’t match the target system. 

POST /nova/v1/mapping/transform-suggest
{
  "field_name": "start_date",
  "incoming_value": "12/01/2025",
  "expected_format": "YYYY-MM-DD"
}

Response:
{
  "recommended_expression": "formatDate(payload.start_date, 'YYYY-MM-DD')",
  "notes": "Convert from MM/DD/YYYY to ISO format"
}

Inline Nova Usage in Workflows

Nova can be invoked as part of a workflow step to enrich or validate data dynamically. 

{
  "step": "nova_validate_mapping",
  "input": {
    "workflow": "new-hire-sync",
    "payload": {
      "first_name": "Jordan",
      "last_name": "Lee",
      "start_date": "12/01/2025"
    }
  }
}

Response:
{
  "status": "ok_with_warnings",
  "warnings": [
    "start_date format does not match expected pattern 'YYYY-MM-DD'"
  ],
  "recommendations": [
    "Apply date normalization before create_user step"
  ]
}

Best Practices

To get the most from Nova in an implementation or support role:

  • Use Nova mapping suggestions at the start of new integrations
  • Enable anomaly detection for critical workflows
  • Review Nova error classifications during incident triage
  • Log Nova recommendations alongside workflow runs
  • Iterate mappings based on Nova’s insight during onboarding

Nova is designed to augment, not replace, human judgment—helping teams reduce time spent troubleshooting and increase confidence in automation.
