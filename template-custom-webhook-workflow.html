Custom Webhook → Workflow

Connect any system that can send HTTP requests to SyncToScale. Define a custom webhook endpoint, map incoming JSON, and orchestrate downstream actions — with Nova validating payloads and suggesting mappings.

Overview

This template lets you integrate tools that don’t have prebuilt connectors. You define a JSON shape, SyncToScale receives HTTPS events, and Nova analyzes payloads to auto-suggest mapping rules.

When to use

  • Internal tools without native integrations
  • Quick prototyping
  • Flexible event ingestion
  • Nova-assisted schema design

What it automates

  • Secure webhook URL creation
  • Payload validation
  • Field mapping
  • Downstream routing
  • Logging with Nova insights

How it works

  1. SyncToScale generates an HTTPS webhook URL.
  2. Your system POSTs JSON payloads.
  3. Nova analyzes keys/values and suggests mappings.
  4. Mapped data routes into workflow actions.
  5. Runs include logs + diagnostics.

Workflow steps

  1. 1 — Incoming HTTP webhook

    Your system sends JSON to the SyncToScale endpoint.

  2. 2 — Validate & parse JSON

    SyncToScale checks structure, required keys, and optional HMAC signatures.

    Nova assists by:

    • Proposing a schema
    • Flagging inconsistencies
    • Recommending required fields

  3. 3 — Transform & map fields

    Map arbitrary payload keys to structured workflow fields.

  4. 4 — Route downstream

    Branch based on payload characteristics.

  5. 5 — Return response

    Send success/error summary back to the caller.

Field mappings

Webhook JSON → SyncToScale

Incoming Key Transform Mapped Field
event_type Lowercase eventType
source_system Trim + lowercase source
timestamp ISO parse occurredAt
payload.user.email Validate email userEmail
payload.account_id Stringify accountExternalId

Sample JSON payloads

Incoming webhook

{
  "event_type": "ticket.created",
  "source_system": "internal_tool",
  "timestamp": "2025-04-03T16:20:00Z",
  "payload": {
    "ticket_id": "TCK-10291",
    "subject": "Cannot access dashboard",
    "priority": "high",
    "user": { "email": "morgan@example.com" }
  }
}

Error handling

Webhook workflows return predictable JSON error structures.

{
  "status": "failed",
  "httpStatus": 400,
  "error": {
    "type": "schema_validation_failed",
    "message": "Missing required key 'event_type'"
  }
}

Nova insights

Set up this template

  1. Select the template in SyncToScale.
  2. Copy your generated webhook URL + secret.
  3. Send example payloads.
  4. Use Nova for schema + mappings.
  5. Add downstream actions.
  6. Activate workflow.
