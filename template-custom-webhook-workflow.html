Custom Webhook → Workflow

Connect any system that can send HTTP requests to SyncToScale. Define a custom webhook endpoint, map incoming JSON, and orchestrate downstream actions — with Nova helping validate payloads and suggest mappings.

Overview

The Custom Webhook → Workflow template lets you integrate tools that don’t have a native connector. You define a JSON payload shape, receive events over HTTPS, and use SyncToScale to map fields, branch logic, and trigger actions — while Nova analyzes payloads and auto-suggests mappings.

When to use this template

  • Integrating an internal tool without an API connector.
  • Prototyping a new integration quickly.
  • Needing a flexible catch-all endpoint.
  • Wanting Nova to validate payloads and suggest mappings.

What this template automates

  • Creates a secure webhook URL.
  • Validates incoming JSON.
  • Maps arbitrary payloads to structured fields.
  • Routes events to downstream actions.
  • Logs payloads and Nova diagnostic notes.

How it works

  1. Create a Custom Webhook template instance in SyncToScale.
  2. SyncToScale generates an HTTPS endpoint + secret.
  3. Your external system sends JSON payloads to that endpoint.
  4. Nova suggests mappings and flags errors.
  5. Data flows into your workflow actions.
  6. All runs include payload + Nova diagnostics.

Workflow steps

  1. Step 1 — Trigger: Incoming HTTP webhook

    The workflow starts when your system POSTs JSON to the unique SyncToScale webhook URL.

  2. Step 2 — Validate & parse JSON

    SyncToScale checks JSON validity, required keys, and optional HMAC signatures.

    Nova assists by:

    • Proposing a JSON schema.
    • Flagging missing or inconsistent fields.
    • Suggesting required/optional keys.

  3. Step 3 — Transform & map fields

    Map JSON keys to structured fields. Nova provides mapping suggestions.

  4. Step 4 — Route to downstream actions

    Branch workflows based on payload content.

    • Create/update CRM or HRIS records
    • Send Slack alerts
    • Write events into logs

  5. Step 5 — Respond to the webhook caller

    Return a JSON success or error summary.

Field mappings

Webhook JSON → SyncToScale event

Incoming key Transform Mapped field
event_type Lowercase eventType
source_system Trim + lowercase source
timestamp ISO parse occurredAt
payload.user.email Validate email userEmail
payload.account_id Stringify accountExternalId

Routing rules

Condition Action Example
eventType = "ticket.created" Create CRM case + Slack alert Support form submission
eventType = "usage.spike" Notify product ops Anomaly detection
eventType = "billing.charge_failed" Create billing task Billing webhook

Sample JSON payloads

Incoming webhook

{
  "event_type": "ticket.created",
  "source_system": "internal_portal",
  "timestamp": "2025-04-03T16:20:00Z",
  "payload": {
    "ticket_id": "TCK-10291",
    "subject": "Cannot access reporting dashboard",
    "priority": "high",
    "user": {
      "email": "morgan@example.com",
      "name": "Morgan"
    },
    "metadata": {
      "plan": "Enterprise",
      "account_id": "ACCT-4390"
    }
  }
}

Normalized event

{
  "eventId": "evt_01J2WEBHOOK",
  "eventType": "ticket.created",
  "source": "internal_portal",
  "occurredAt": "2025-04-03T16:20:00Z",
  "accountExternalId": "ACCT-4390",
  "userEmail": "morgan@example.com",
  "data": {
    "ticketId": "TCK-10291",
    "subject": "Cannot access reporting dashboard",
    "priority": "high"
  }
}

Response example

{
  "status": "accepted",
  "runId": "run_01J2WEBHOOKFLOW",
  "novaSummary": "Ticket created and Slack alert sent."
}

Error handling

Webhook workflows return structured JSON errors for predictable integration behavior.

{
  "runId": "run_01J2WEBHOOKFLOW",
  "status": "failed",
  "httpStatus": 400,
  "error": {
    "type": "schema_validation_failed",
    "message": "Required field 'event_type' missing",
    "novaHints": [
      "Verify required keys",
      "Use Preview mode to validate payloads"
    ]
  }
}

Nova insights

  • Suggests mapping rules from sample data.
  • Identifies missing or inconsistent fields across runs.
  • Groups similar payloads into suggested flow branches.
  • Highlights high-error external systems.

Set up this template

  1. Select Custom Webhook → Workflow in Templates.
  2. Copy the webhook URL & secret into your external system.
  3. Send example payloads in Preview mode.
  4. Use Nova to define schema & mappings.
  5. Add downstream actions (Slack, CRM, etc.).
  6. Activate the workflow once validated.

Template summary

  • Trigger: Webhook (POST)
  • Runtime: Real-time
  • Owner: Dev · Ops

Best practices

  • Enable HMAC or token validation.
  • Use Nova to lock down schema.
  • Return 202 for async workflows.
