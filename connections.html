Connections

Connections link SyncToScale to your external applications. Each connection is securely stored, validated, monitored, and refreshed to ensure workflows run smoothly across systems.

Overview

Connections are the foundation of every workflow. Whether you're using OAuth 2.0, API keys, or service accounts, SyncToScale ensures each connection remains valid and ready for execution.

Creating a Connection

A new connection begins by selecting an app (e.g., Slack, HubSpot, BambooHR) and initiating an OAuth or API key authentication flow. SyncToScale automatically encrypts and stores all credentials. 

POST /connections
{
  "provider": "slack",
  "auth_type": "oauth",
  "scopes": ["users:read", "chat:write"]
}

After successful authentication, the connection receives a unique ID used in workflows.

Connection States

Each connection has a clear state for easy troubleshooting:

  • Healthy — Valid token, correct scopes, no issues detected.
  • Warning — Near rate limits, missing optional scopes, or minor issues.
  • Needs Attention — Token expired or refresh failed; user action required.
  • Error — Invalid credentials, revoked access, or severe upstream outage.
GET /connections/{id}
{
  "id": "conn_abc123",
  "provider": "slack",
  "status": "healthy",
  "last_validated": "2025-01-10T15:22:00Z"
}

Health Checks

SyncToScale performs ongoing health checks to prevent workflow failures. These checks include:

  • Token validity
  • Provider API availability
  • Permission scope verification
  • Endpoint response structure
  • Rate-limit and throttling detection
GET /connections/{id}/health
{
  "status": "warning",
  "details": ["Rate limit threshold reached"],
  "last_checked": "2025-01-19T08:32:11Z"
}

Token Refresh Logic

Connections authenticated via OAuth refresh automatically when nearing expiration. If a refresh attempt fails due to revoked access or insufficient scopes, SyncToScale sets the connection to Needs Attention. 

POST /connections/{id}/refresh
{
  "refresh_token": "1//0gH3...",
  "provider": "google"
}

A successful refresh updates the access token and timestamp.

Reconnecting

If a connection enters an error state, users can easily reconnect by re-authenticating. Previous mappings and workflows remain intact. 

POST /connections/{id}/reconnect
{
  "redirect_url": "https://synctoscale.me/oauth/callback"
}

Once reconnected, workflows resume from the next available run.

Connection-Level Troubleshooting

Common issues encountered during implementation and support:

  • Insufficient Scopes — Workflow action requires scopes the admin did not grant.
  • Revoked Access — User removed app access from the provider dashboard.
  • Incorrect API Key — Key expired or was rotated without updating SyncToScale.
  • Provider Outages — Slack, HubSpot, etc. return 500/503 errors impacting health checks.
  • Rate Limit Lockout — Too many requests cause temporary suspension.
  • Region Restrictions — Provider API only available in certain regions or tenants.

SyncToScale surface these issues in monitoring logs and alerts, helping teams respond quickly during onboarding or production syncs.
