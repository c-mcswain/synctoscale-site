Authentication

SyncToScale supports OAuth 2.0, API key authentication, and scoped access. This guide explains how connections are created, refreshed, validated, and monitored.

Overview

Every integration in SyncToScale begins with a secure, validated connection. Once authenticated, SyncToScale performs automated health checks, token refresh logic, and permission validation to ensure every workflow run has access to required resources.

OAuth 2.0 (Recommended)

POST /auth/oauth/start
{
  "provider": "google",
  "redirect_url": "https://synctoscale.me/oauth/callback"
}

Token Response

{
  "access_token": "ya29.a0AW...",
  "refresh_token": "1//0gH3...",
  "expires_in": 3600,
  "scope": "users.read calendar.readonly"
}

SyncToScale stores encrypted tokens and refreshes them automatically. If a refresh fails, the connection enters a Needs Attention state.

API Key Authentication

POST /auth/api-key
{
  "api_key": "sk_live_abc123",
  "provider": "legacy_crm"
}

SyncToScale validates the key by performing a lightweight test request to confirm scope and access.

GET Connection Health

GET /connections/{id}/health
{
  "status": "healthy",
  "last_checked": "2025-01-18T12:44:00Z",
  "warnings": []
}

Error States

  • Expired Token ‚Äî Refresh failed, requires reconnect.
  • Insufficient Permissions ‚Äî Workflow requires scopes not granted.
  • Invalid Credentials ‚Äî API key or token no longer valid.
  • Provider Outage ‚Äî Upstream service unreachable.
  • Rate Limit Lockout ‚Äî Temporary hold triggered by remote provider.

Best Practices

  • Use OAuth whenever possible
  • Grant the minimum required scopes
  • Rotate API keys every 90 days
  • Monitor connection warnings via logs
